02 GETTING STARTED
~2 MIN

Install

Drop Void UI into any Next.js 14+ project. Ships with Tailwind config and CSS variables — no runtime, no theme provider.

TERMINALzsh 
# 01 · install
pnpm add @void/ui

# 02 · import styles (app/layout.tsx)
import "@void/ui/tokens.css";

# 03 · use
import { Button } from "@void/ui";
TAILWINDtailwind.config.js 
const void_ui = require("@void/ui/tailwind");

module.exports = {
  presets: [void_ui],
  content: [
    "./app/**/*.{ts,tsx}",
    "./node_modules/@void/ui/**/*.js",
  ],
};

04 PATTERNS
ARCHITECTURE

Architecture

Four non-negotiable patterns. Every component in Void UI follows all four.

01
"use client" Boundary

Interactive components carry the directive. Layout and wrapper components stay as server components — a thin client shell around server-safe primitives.

use client RSC SAFE SERVER COMPONENTS 
// button.tsx
"use client"

export function Button({ ... }) {
  return <button data-state={state} />
}
02
Compound Components

Context-backed sub-components. Consumers compose rather than configure — no prop drilling, no render props, no magic strings.

React.Context COMPOSABLE 
<Select>
  <Select.Trigger />
  <Select.Content>
    <Select.Item value="eu-1" />
  </Select.Content>
</Select>
03
Headless + Styled Layer

Logic lives in hooks (useDialog, useCombobox). The visual layer is a separate thin skin — consumers can restyle without touching behaviour.

HOOKS FIRST RADIX MODEL RESTYABLE 
// headless hook
const { open, toggle } = useDialog()

// styled layer consumes it
<DialogRoot state={open} />
04
forwardRef + asChild

Every component forwards refs. asChild via Radix Slot lets consumers swap the underlying DOM element — zero wrapper divs.

POLYMORPHIC SLOT PATTERN NO WRAPPERS 
<Button asChild>
  // renders as <a>, not <button>
  <Link href="/docs">Read docs</Link>
</Button>

CON CONVENTIONS
CVA · VARIANTS

CVA Variants

class-variance-authority drives all component variants. Type-safe props, zero runtime cost, composable with cn().

DEFINITIONbutton.variants.ts 
import { cva } from "class-variance-authority";

export const buttonVariants = cva(
  "inline-flex items-center gap-2 uppercase tracking-wide",
  {
    variants: {
      variant: {
        primary:   "bg-fg text-bg border-fg",
        secondary: "bg-transparent text-fg border-fg",
        ghost:     "border-transparent text-fg-muted",
        accent:    "bg-accent text-accent-fg border-accent",
      },
      size: {
        sm: "h-[26px] px-2.5 text-[10px]",
        md: "h-[32px] px-3.5 text-[12px]",
        lg: "h-[42px] px-5 text-[13px]",
      },
    },
    defaultVariants: { variant: "primary", size: "md" },
  }
);
USAGE · TYPE-SAFE PROPSpage.tsx 
import { Button } from "@void/ui";

// TS autocompletes variant + size
<Button variant="accent" size="lg">
  Deploy
</Button>

// cn() lets consumer classes win
<Button className="w-full mt-4">
  Full width
</Button>

// asChild renders as <a>
<Button asChild variant="ghost">
  <Link href="/docs">Docs</Link>
</Button>
CON CONVENTIONS
data-state

data-state

Every component exposes its state as a data-state attribute. Style with CSS, animate with Tailwind, or read with JS — no prop drilling needed.

ATTRIBUTE MAPdata-state
BUTTON
data-state="loading"
data-disabled
data-variant="accent"
[data-state="loading"] → show spinner, lock pointer-events
DIALOG · POPOVER
data-state="open"
data-state="closed"
[data-state="open"] → entry animation; closed → exit animation via tailwind-animate
INPUT · SELECT
data-state="error"
data-state="success"
data-disabled
[data-state="error"] → red border + shake animation via @keyframes
ACCORDION · TABS
data-state="open"
data-state="closed"
data-state="active"
[data-state="open"] .acc-arrow → rotate(180deg). No JS class toggling needed. 
/* target data-state in CSS — no className juggling */
[data-state="open"] { animation: slideIn 180ms var(--ease-snap); }
[data-state="closed"] { animation: slideOut 120ms var(--ease-snap); }
[data-disabled] { opacity: .4; pointer-events: none; }
[data-state="error"] input { border-color: oklch(64% 0.24 25); }
